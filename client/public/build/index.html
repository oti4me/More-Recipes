    More Recipes API Documentation

    Welcome to the More Recipes API! You can use our API document to access our Recipes API endpoints, which is used to add and manage recipe ideas from parts of the world. These recipe collections are accessable as stated on this document.

    Development

    The application was developed with:

    1. NodeJs: Node.js is used as the server framework for serving the application
    2. Express: Express is a node.js server that is used to handle routing the endpoints.
    3. Postgresql database: For data persistance
    4. Sequelize as the ORM for interacting with posgresql database

    Installation

    Follow the steps below to setup a local development environment. First ensure you have Postgresql installed, and a version of NodeJs equal or greater than v6.8.0.

    1. Clone the repository from a terminal git clone https://github.com/oti4me/More-Recipes.git.
    2. Move into the project directory using cd More-Recipes
    3. Add .env file and add any preferred postgresql using the .env-example file.
    4. Install project dependencies by using npm install command on your terminal
    5. Start the express server using npm run dev command on your terminal.
    6. Visit localhost:3000 to test the endpoints

    Errors

    The More Recipes API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request validation returned error.
    401 Unauthorized -- Your API key is wrong.
    404 Not Found -- The specified resource could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
